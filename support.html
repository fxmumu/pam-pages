常见问题与快速解决

1) 无法加载原图 / 出现 PHPhotosErrorDomain 3303

这种情况常见于照片存储在 iCloud 而设备未下载完整原件的情形。排查步骤：

  • 确保设备网络连通并允许应用使用网络（Wi‑Fi / 蜂窝）。
  • 在系统“设置 → 照片”中查看是否启用了“优化 iPhone 存储”；如需要，临时切换为“下载并保留原件”以测试。
  • 在 App 内长按/打开该照片并在系统照片应用中查看是否可下载原始文件。
  • 开发者提示：若您是开发者，请检查照片请求 API 中是否允许网络访问（PHImageRequestOptions.isNetworkAccessAllowed）。

2) 图片缩略图 / 预览不显示或质量低

  • 这是由于请求的目标尺寸或缓存策略决定的；退出并重新打开相册有时可触发刷新。
  • 在“设置 → 隐私与安全 → 照片”中确认本应用被授予了照片访问权限。

3) 购买/订阅问题（购买失败 / 恢复失败 / 收据相关）

  • App 内购买通过 Apple App Store 完成，购买和退款请先查看 Apple 的订单与收据信息。
  • 要恢复购买：在 App 的设置页使用“恢复购买”按钮（或在 App Store 的购买记录中查找订单）。
  • 如果仍有问题，请把购买凭证截图或 App Store 订单 ID 发给我们，我们将协助核查。

4) 为什么搜索结果里可能会出现毫不相关的结果？

PicSeek 的搜索核心是“相似度匹配”：系统会把照片内容（以及可用的文本信息）转换成一串用于计算相似度的特征，并在大量照片里找出“看起来最像”的一些结果。由于相似度计算是近似的，它有时会把某些共同特征误判为相关，例如：

另外，相似度搜索通常会按相似度从高到低排序，并返回“最相似的前 N 条结果”（N 由系统设定或由一次搜索想展示的条数决定）。这并不代表这 N 条都一定与您的搜索意图强相关：当照片库里缺少足够接近的内容时，排名靠后的结果可能会显得不相关，但它们仍然是当前库里“相对更像”的候选。

  • 颜色/光线/构图相似（比如都是白底、同样的明暗对比）。
  • 场景元素相似（比如都有屏幕、文档、人物、建筑等）。
  • 查询词过短或语义过泛，系统只能“猜”你想找的方向。

建议：尝试输入更具体的关键词（更长的描述往往更准确），或换一个更贴近目标内容的说法重新搜索；也可以先用相册/归集功能把候选范围缩小后再检索。

5) 为什么搜索暂不支持“按时间范围”筛选？

目前搜索主要针对“内容相似度”做排序与召回，暂未提供时间范围的筛选入口。时间范围筛选需要在检索阶段引入额外的过滤条件，并与当前的相似度检索流程配合（否则会出现结果不稳定或性能下降）。我们会根据使用反馈评估是否在后续版本加入该能力。

6) 搜索结果较少？请确认照片解析进度为 100%

  • PicSeek 需要先对照片进行解析/索引后，才能参与搜索。
  • 如果您发现搜索结果偏少，请到 App 的“设置”页查看照片解析进度，确保解析进度达到 100%。
  • 解析过程中请尽量保持 App 在前台或允许其继续运行，并保持设备电量与存储空间充足。
